---
title: "Steamtools Quickstart: First Run to Managing Manifests"
description: "Launch stcli.exe, configure your Steam path, load your first Lua manifest, and start browsing and managing your game manifest library."
---

This guide walks you through launching Steamtools for the first time and performing the most common tasks: loading a Lua manifest, browsing your library, and acting on a game.

<Steps>
  <Step title="Launch stcli.exe">
    Double-click `stcli.exe`. The app opens to the **Setup** screen on the very first run — or any time the stored Steam path can no longer be found.
  </Step>

  <Step title="Set your Steam path">
    On Windows, Steamtools reads the Steam installation path from the registry automatically. The path field is pre-filled with the value found under `HKLM\SOFTWARE\WOW6432Node\Valve\Steam`.

    If the field is empty or incorrect, type the path manually or click the field to open a folder picker. The default Steam location is:

    ```text
    C:\Program Files (x86)\Steam
    ```

    Once the path is filled in, click **Validate**. Steamtools checks that `steam.exe` exists at that location. If the check passes, a confirmation dialog appears and the app moves to the **Main Menu**. If it fails, correct the path and try again.

    <Note>
      The path must point to the Steam installation folder itself — the one that contains `steam.exe` — not a subfolder inside it.
    </Note>
  </Step>

  <Step title="Explore the main menu">
    After validation you land on the **Main Menu**. The central panel shows a scrollable grid of game cards. Each card displays the Steam header image for that app ID. The right panel shows actions for whichever card you have selected.

    If the grid is empty, no `.lua` files are present in `Steam\config\stplug-in\` yet. Load one in the next step.
  </Step>

  <Step title="Load a Lua manifest">
    Click **Load lua** in the toolbar. A file picker opens filtered to `.lua` files. Select one or more manifest files and confirm.

    Steamtools copies each file into `Steam\config\stplug-in\` using the file's stem as the name. The stem must be the Steam App ID of the game — for example, a file named `480.lua` maps to app ID `480` (Portal).

    After loading, the game grid refreshes automatically and fetches header images from the Steam store API.

    <Warning>
      Name your `.lua` files by their Steam App ID (e.g. `480.lua`). Files with non-numeric names are skipped and logged as errors.
    </Warning>
  </Step>

  <Step title="Search your library">
    Use the search bar at the top of the window to filter the game grid by name or app ID. Type a game name to match by prefix, or type a numeric app ID for an exact match.

    <Tip>
      Press **Tab** anywhere in the app to jump focus directly to the search bar without reaching for the mouse.
    </Tip>

    Click the **✕** button next to the search bar to clear the filter and show all manifests again.
  </Step>

  <Step title="Act on a game">
    Click any game card in the grid to select it. The right panel updates with the game's name and app ID, and shows the available actions:

    - **View** — Opens a popup showing the game's App ID details.
    - **Install** — Triggers Steam's native install dialog for the selected app ID (only shown when the game is not installed).
    - **Uninstall** — Triggers Steam's native uninstall dialog (only shown when the game is installed).
    - **Remove** — Deletes the `.lua` manifest file from `Steam\config\stplug-in\` and removes the game card from the grid.
  </Step>
</Steps>
